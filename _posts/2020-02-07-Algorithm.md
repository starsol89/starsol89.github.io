# 알고리즘

## 알고리즘이란?

컴퓨터를 이용하여 주어진 과제를 해결하기 위한 처리 절차. 즉, 문제를 해결하기 위한 처리 절차이다.

ex) 현실 세계의 다양한 과제의 해결에 있어서도 알고리즘의 개념이 사용된다. = 음식 요리법

- 정당성 - 주어진 과제에 대해 올바른 결과를 반환 해야한다. 이때 "단정문- 실행 순서 중 임의의 위치에 서서 충족해야 하는 조건이 성립 하는지의 여부를 체크하게 된다."

- 정지성 - 어떤 조건의 입력 값이 주어지더라도 정해진 시간 내에 반드시 정상적인 종류를 보장 해야한다.

알고리즘에는 다양한 종류가 있다.

1. 기술 계산

   - 유클리드 호제법(최대공약수)
   - 가우스 소거법(방정식)
   - 사다리꼴의 법칙(정적분)
   - 데이크스트라 알고리즘(최적 경로)
   - 에라토스테네스의 체(소수)

2. 정렬(Sort) - 1줄로 늘어선 데이터를 작은순서(오름차순) 또는 큰 순서(내림차순)로 정렬하는 알고리즘 입니다.

   - 단순 선택 정렬
   - 단순 교환 정렬(버블 정렬)
   - 단순 삽입 정렬
   - 셸 정렬
   - 병합 정렬
   - 퀵 정렬

3. 검색(Serch) - 많은 양의 데이터 중에서 원하는 데이터를 찾아내기 위한 알고리즘 입니다.

   - 선형 검색(리니어 서치)
   - 이진 검색(바이너리 서치)

4. 문자열 패턴 매칭 - 문자열 중에서 지정한 문자열의 패턴(부분 문자열)과 일치하는 부분을 찾아내기 위한 알고리즘 입니다.

   - 단순 문자열 일치
   - KMP 알고리즘
   - BM 알고리즘

## 데이터

알고리즘을 고안 할 때 데이터(다양한 정보)가 필요하다. 데이터란 다양한 정보를 표현 한 것이다.

데이터에는 타입이 존재한다.

1. 정수타입 - 정수(소수점이 없는 값) ex) 0, 1, 100, 9999, -123
2. 실수타입 - 실수(소수점을 포함한 값) ex) 1.23, 3.141592, -99.9
3. 문자타입 - 문자 ex) A, B, C, b, g, 가, 나
4. 문자열타입 - 문자열 ex) 세종대왕, KOREA, 알고리즘
5. 논리타입 - '참', '거짓'으로 구분한다. ex) true, false

값에 해당하는 데이터는 알고리즘으로 옴길때, 숫자와 문자, 문자열을 구분해줘야한다.

일반적으로 숫자는 그냥 1,3,123, -123, 작성하지만 문자는 '가' 문자열은 "가" 이렇게 구별해줘야한다. 즉, 작은따옴표, 큰따옴표로 말이다.

## 변수

알고리즘에서 데이터를 조작할 때 조작한 데이터를 저정할 공간이 필요한데 그 공간이 '변수'다.

즉, 데이터(값)를 넣어 두는 상자이며, 데이터(값)을 넣는 작업을 '대입한다'고 한다.

변수는 '변수명'으로 구분지어 준다. 이러는 이유는 변수의 용도가 무엇인지를 표시하고, 다른 변수들과 구별하기 위한 수단이다.

다만 변수명을 붙일 때는 나름의 법칙이 존재한다.

1. 변수명은 고유해야 한다.
2. 숫자만 사용한 변수명이나, 숫자로 시작하는 변수명은 쓸 수 없다.

변수에도 데이터 타입이 존재한다.

1. 정수 타입
2. 실수 타입
3. 문자 타입
4. 문자열 타입
5. 논리 타입

## 배열

만약 전교생의 시험 결과(점수)나 1년간의 일일 입장객 수 또는 상품의 월별 매출액 처럼 많은 양의 데이터를 저장하고 유지하기 위해서 어떤 방법을 써야 할까?

답은 배열이다.이러한 데이터를 처리하기 아주 좋다.

다만 원칙으로 배열에 담을 각각의 데이터들은 반드시 같은 종류여야만 한다.

좀 더 자세히 말하면 변수라는 상자를 일직선 상에 빈틈없이 늘어 놓은 것이 배열이고, 그러한 점에서 배열은 크기가 같은 서랍들이 들어찬 옷장과 ㅣ비슷하다. 또한 데이터 타입이 동일한 변수를 늘어 놓은 것이기 때문에, 배열 자체에도 데이터 타입이 존재한다. 이 데이터 타입은 배열을 만들 때 지정한다.

즉, '같은 종류의 데이터(값)를 대량으로 저장하는 여러 개의 상자들을 1줄로 나열한 것'

변수 처럼 배열도 '배열명'을 작성해서 다른 것과 구분 되고 선택이 쉽도록 해준다.

배열에서 상자(변수) 하나 하나를 '배열 요소' 라고 부르며, 배열에 존재하는 상자의 개수는 '배열 요소 수'라고 말한다. 1번째 배열 요소 부터 마지막 배열 요소까지 일련번호를 붙여서 어떤 배열 요소에 데이터를 넣을지 선택하거나, 어떤 배열 요소의 데이터를 참조 할지를 지정한다.

그렇게 연속 된 번호를 '요소 번호' 라고 한다.

이때 요소 번호를 사용해서 알고리즘을 작성할 때는 배열의 1번째 요소의 번호가 '0' 부터 시작하는 것과 '1' 부터 시작하는 경우가 있다는 점에 반드시 주의 해야한다.

전자를 'BASE 0', 후자를 'BASE 1' 이라고 한다.

배열의 특정 요소에 접근할 때, 배열명과 요소 번호를 조합해서 가리키는 것이 일반적이다. 또한 요소 번호를 배열의 '첨자' 라고 한다.

또한 배열에는 일직선 상에 나열한 사물함의 모습을 한 배열을 '1차원 배열' 이라고 한다. 배열 요소를 요소 번호 하나로 가리킬 수 있기 때문이다.

반면에 변수를 가로세로 빈틈없이 직사각형 모양으로 나열한 배열도 존재하는데, 이것을 '2차열 배열' 이라고 부른다.

또 변수를 가로, 세로, 높이 빈틈없이 직육면첵 모양으로 나열한 배열도 존재하는데, '3차원 배열' 이라고 부른다.

이 모두를 합하여 '다차원 배열' 이라고 부른다.

특히나 '2차원 배열'은 '행'과 '열'에 대해 각각 '0'(또는 '1')로 시작하는 요소 번호를 붙여서 그 교차점에 있는 배열 요소를 가리킨다.

2차원 배열의 특정 요소에 접근할 때에는 배열 이름과 요소 번호를 조합하여 작성하는 것이 일반적인데,

예를들어, 배열명[행 요소 번호][열 요소 번호], 배열명(행 요소 번호)(열 요소 번호) 혹은 배열명[행 요소 번호, 열 요소 번호], 배열명(행 요소 번호, 열 요소 번호)로 가르킨다.

만약 행 요소 번호가 2, 열 요소 번호가 6인 2차원 배열 ARRAY의 요소를 가리킨다면, ARRAY[2][6] ARRAY(2)(6) or ARRAY[2,6], ARRAY(2,6)과 같이 첨자 2개를 사용하면 된다.

## 문자열

알고리즘에서 문자와 문자열 처리는 다소 특수한 케이스이다. 그 이유는 컴퓨터에서 문자와 문자열을 처리하는 독특한 방식에 의존하고 있기 때문이다.

0과 1 이란 2진수 숫자의 조합을 사용하여 현실세계의 정수 값이나, 실제값과 같은 수치를 컴퓨터에서 표현한다. 문자 역시 컴퓨터 내부적으로는 숫자(문자코드)로 표현하게 된다.

가령 'A'는 65 'B'는 66 'C'는 67 그리고 '0'은 48 '+'는 43 등등으로 표현되기로 약속 되어 있다.

그런데 컴퓨터 세계에서 '문자열'이란 방금 전에 문자 코드 시스템의 '연속된 문자들'로 표현하도록 약속이 되어 있다. 따라서 '문자열' 이란 '문자'를 일직선 상에 빈틈없이 나열한 정보라고 정리 할 수 있다.

즉, '문자열은 각 요소에 문자가 저장된 문자 배열이다.' 라고 정의 할 수 있다.

예를들어 문자열 "ABC"는 문자 'A', 문자 'B', 문자 'C'를 배열 요소로 저장한 다음 배열로 표현하는 것이다.

즉, 문자 타입 데이터를 저장할 수 있는 배열이 있고, 그 1번째 요소에 문자 'A', 두번째 요소에 문자 'B', 3번째 요소에 문자 'C'가 저장되어 있는 상태가 문자열 "ABC"라고 할 수 있다.

배열에 저장된 문자 데이터만으로는 그 배열의 몇번째 요소에서 문자열이 끝나는지 알 수 있는 방법이 없다. 이 방법을 알기 위해서 2가지 방법이 존재한다.

1. 문자열 길이 변수를 준비하는 방법 (len)
2. 문자열 끝에 '보초 값'을 저장하는 방법

1은 배열 속 문자열의 길이를 정수 타입 변수에 저장하는 방법이다.

2는 문자열의 구성 문자로 절대로 표시되지 않는 문자 코드를 배열의 문자열 끝 요소 뒷 부분에 저장하는 방법이다. 그러한 문자 코드를 '보초 값' 이라 한다. 보초 값으로 0(숫자)를 사용하는 것이 일반적이다.예를 들어 "KOREA" 라는 문자열이 있다면 str[0]~str[4]까지의 len인데, str[5]에 값 0을 저장해 둔다는 말이다.

## 자료구조

대량 데이터를 효율적으로 관리하기 위한 메커니즘이 자료구조 이다.

알고리즘에서 사용되는 자료구조에는 다양한 종류가 있다.

1. 배열 - 데이터를 빈틈없이 나열한 자료구조이며, 1차원 배열, 2차원 배열, 3차원 배열이 존재한다.
2. 리스트 - 데이터를 순서대로 나열한 자료구조이며, 배열과 같이 차례대로 나열한 데이터를 관리한다. 그러나 데이터들이 화살표로 서로 연결되어 있어 데이터들이 떨어진 장소에 위치해도 된다는 점이 배열과 다르다.
3. 스택 - 책상 위에 책을 쌓듯 데이터를 관리하는 자료구조이며, 쌓여 있는 책은 가장 위쪽 부터 꺼낼 수 밖에 없는데 이와 마찬가지로 데이터를 넣는 순서와 반대의 순서로 데이터를 꺼내는 데이터 관리 방법이다.
4. 큐(대기 행렬) - 계산대에서는 먼저 줄을 선 손님부터 차례대로 계산합니다. 이와 마찬가지로 데이터를 넣은 순서대로 데이터를 꺼내는 데이터 관리 방법이다.
5. 트리(나무 구조) - 나무가지가 2개, 3개, 로 갈라지고 그 갈라진 끝에서 2개, 3개 다시 나뉘듯 퍼져 나가는 자료구조이다.

### 책처럼 쌓이는 자료구조가 스택!

STACK의 뜻은 '쌓다'라는 뜻이 있다. 이처럼 데이터를 쌓아서 관리한다.

예를들어 자동차 폐차장에서 구겨진 자동차를 꽤 높은 위치까지 쌓았다면, 폐기처분 할 때 중간에서 부터 꺼내서 폐차 시킬 수 없듯, 위에서 순서대로 꺼내서 폐차 시켜야한다. 이 처럼 스택도 그와 마찬가지의 구조를 지닌다.

즉, 스택을 사용한 데이터 관리법에서는

1. 데이터를 넣는(쌓는) 작업을 푸시(PUSH)
2. 데이터를 꺼내는 작업을 팝(POP)

이라고 한다.

또한 스택 처럼 마지막에 입력된 데이터가 먼저 출력되는 특징을 갖는 데이터 관리 방식을 LIFO(Last In, First Out) 또는 FILO(First In, Last Out)라고 부른다.

### 계산대앞에 줄을 서듯 대기하는 자료구조가 대기 행렬(큐)!

QUEUE란 먼저 입력한 데이터가 먼저 출력되는 특징을 가진 자료구조로, 큐 또는 대기 행렬이라 부른다.

예를들어

1. 슈퍼마켓에서 계산을 하기 위해 계산대에 줄을 서서 기다리는 사람들
2. 돈을 인출하기 위해 현금 인출기에 줄을 서서 기다리는 사람들
3. 역 앞에서 버스를 타기 위해 줄을 서서 기다리는 사람들

또 새로운 대기자는 그 줄의 가장 끝으로 가서 줄을 서게 된다. 또 맨 처음 줄 선 사람의 용무가 끝나면 그 다음 사람이 다시 첫번째 대기자가 되게 된다. 이처럼 반드시 먼저 줄을 선 사람부터 순서대로 처리되는 자료구조가 큐(대기 행렬)이라 한다.

입력한 데이터가 먼저 출력되는 특징을 갖는 데이터 관리 방법을 FIFO(Fast In, Fast Out) 또는 LILO(Last In, Last Out)라고 한다.
